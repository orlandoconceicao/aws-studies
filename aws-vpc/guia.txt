GUIA COMPLETO – AWS VPC
Virtual Private Cloud
============================================================


1) O QUE É VPC?
------------------------------------------------------------

VPC (Virtual Private Cloud) é uma rede virtual privada
dentro da AWS.

Ela permite criar um ambiente isolado para executar
EC2, RDS, Load Balancer, Lambda e outros serviços.

Pense nela como o “data center privado”
da sua aplicação na nuvem.


------------------------------------------------------------
2) COMPONENTES PRINCIPAIS
------------------------------------------------------------

VPC
Rede principal. Define o bloco CIDR.

SUBNET
Divisão da VPC.
Pode ser:
- Pública
- Privada

INTERNET GATEWAY (IGW)
Permite comunicação com a internet.

ROUTE TABLE
Define para onde o tráfego será direcionado.

NAT GATEWAY
Permite que subnets privadas acessem a internet
sem ficarem expostas.


------------------------------------------------------------
3) CIDR (BASE DA REDE)
------------------------------------------------------------

Exemplo:

10.0.0.0/16

Significa:
65536 endereços IP disponíveis.

Subdivisão:

10.0.1.0/24
256 endereços IP disponíveis.

Planejamento de CIDR é essencial
para arquitetura escalável.


------------------------------------------------------------
4) SUBNET PÚBLICA VS PRIVADA
------------------------------------------------------------

Subnet Pública:
- Possui rota para Internet Gateway
- Pode ter IP público
- Ideal para Load Balancer e EC2 Web

Subnet Privada:
- Sem acesso direto à internet
- Mais segura
- Ideal para RDS e serviços internos


------------------------------------------------------------
5) ARQUITETURA PROFISSIONAL
------------------------------------------------------------

Internet
   ↓
Internet Gateway
   ↓
VPC (10.0.0.0/16)
   ↓
----------------------------------
| Subnet Pública   → EC2 Web     |
| Subnet Privada   → RDS         |
----------------------------------

Separação de camadas:
- Camada Web (pública)
- Camada Banco (privada)


------------------------------------------------------------
6) COMO APLICAR NA PRÁTICA (PASSO A PASSO)
------------------------------------------------------------

1. Criar VPC
   - CIDR: 10.0.0.0/16

2. Criar Subnet Pública
   - CIDR: 10.0.1.0/24
   - Habilitar auto-assign public IP

3. Criar Internet Gateway
   - Associar à VPC

4. Criar Route Table Pública
   - Adicionar rota:
     0.0.0.0/0 → Internet Gateway

5. Associar Route Table à Subnet Pública

6. Criar Subnet Privada
   - CIDR: 10.0.2.0/24
   - Sem rota para internet

7. Criar RDS dentro da Subnet Privada

Resultado:
- EC2 acessível via internet
- RDS protegido internamente


------------------------------------------------------------
7) BOAS PRÁTICAS DE MERCADO
------------------------------------------------------------

- Nunca colocar banco em subnet pública
- Usar múltiplas Availability Zones (Multi-AZ)
- Criar VPC separada para produção
- Separar ambiente DEV e PROD
- Usar Security Groups como firewall
- Planejar CIDR pensando em crescimento


------------------------------------------------------------
8) ERROS COMUNS
------------------------------------------------------------

- Usar VPC padrão da AWS em produção
- Colocar banco de dados público
- Não configurar rota corretamente
- Liberar portas desnecessárias (0.0.0.0/0)
- Misturar ambientes no mesmo bloco CIDR


------------------------------------------------------------
9) INTEGRAÇÃO COM OUTROS SERVIÇOS
------------------------------------------------------------

VPC + EC2 → Hospedagem de aplicação
VPC + RDS → Banco isolado
VPC + IAM → Controle de acesso
VPC + Load Balancer → Alta disponibilidade
VPC + Lambda → Serverless privado


------------------------------------------------------------
