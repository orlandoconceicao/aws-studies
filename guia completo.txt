AWS EC2 – Elastic Compute Cloud
============================================================

------------------------------------------------------------------------------
O QUE É?
------------------------------------------------------------------------------

Serviço de máquinas virtuais na AWS.
Permite criar servidores sob demanda com controle total
sobre sistema operacional, CPU, memória e rede.

------------------------------------------------------------------------------
CONCEITOS IMPORTANTES
------------------------------------------------------------------------------

Instance Type (t3.micro, m5, c5, g4, etc)

AMI (Amazon Machine Image)

Security Group

Elastic IP

Auto Scaling Group

Elastic Load Balancer

EBS (Elastic Block Store)

------------------------------------------------------------------------------
BOAS PRÁTICAS
------------------------------------------------------------------------------

Nunca expor banco em EC2 pública

Usar Security Group restritivo

Utilizar Auto Scaling para alta disponibilidade

Usar IAM Role ao invés de Access Key

Criar snapshots periódicos

------------------------------------------------------------------------------
QUANDO USAR
------------------------------------------------------------------------------

APIs tradicionais

Sistemas legados

Aplicações que precisam controle total do servidor

3) AWS LAMBDA – Serverless
============================================================

------------------------------------------------------------------------------
O QUE É?
------------------------------------------------------------------------------

Execução de código sem gerenciar servidor.

------------------------------------------------------------------------------
CARACTERÍSTICAS
------------------------------------------------------------------------------

Escala automática

Paga por execução

Integra com SNS, SQS, API Gateway, DynamoDB

------------------------------------------------------------------------------
LIMITAÇÕES
------------------------------------------------------------------------------

Timeout máximo de 15 minutos

Cold start

Limite de memória

------------------------------------------------------------------------------
BOAS PRÁTICAS
------------------------------------------------------------------------------

Usar IAM Role

Manter funções pequenas

Separar lógica de negócio

Monitorar com CloudWatch Logs

------------------------------------------------------------------------------
QUANDO USAR
------------------------------------------------------------------------------

APIs serverless

Processamento orientado a eventos

Automação

Webhooks
